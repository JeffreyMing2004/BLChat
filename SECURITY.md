# Security Policy

## 报告安全问题

如果你发现了安全相关问题（例如：密钥泄露、请求签名实现缺陷、可被利用的崩溃或远程注入风险），请不要公开发布 Issue 或在群聊中扩散。

建议做法：

- 在 GitHub 通过私信或邮件联系维护者（如仓库主页提供）
- 提供最小复现、影响范围、建议修复方式

## 敏感信息范围

以下内容视为敏感信息：

- 哔哩哔哩开放平台 Access Key / Access Secret
- App ID 与身份码（roomCode）在组合使用时可能产生安全风险
- 任何会话 token、Cookie、抓包内容

## 日志与截图

- 不要在日志或截图中暴露 Access Key / Secret
- 如需反馈问题，建议打码后再提供
